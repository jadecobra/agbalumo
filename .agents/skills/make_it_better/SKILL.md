# Skill: Make It Better (Refactor)
name: make_it_better
description: |
  Optimize, clean, and DRY the working codebase without breaking the existing test suite.
  A 10x refactor MUST result in lower or equal cognitive complexity (gocognit)
  and zero new duplication (dupl).
---
## Objective
As the BackendEngineer, refactor the Green-phase implementation into its final 10x state.

## Rules of Engagement
- **The Specification**: You MUST read and address every item in `REF_TODO.md` (generated by the ChiefCritic).
- **The Red-Green Guard**: Do NOT change test inputs/outputs or functional behavior. Tests must stay green.
- **Complexity Guard**: You MUST run `scripts/critique.sh` to verify that cognitive complexity remains within strict limits (threshold < 10).
- **DRY Policy**: A strict 15-line limit applies to all code blocks. Any code block >= 15 lines MUST be extracted to internal utilities if duplicated.
- **Verification Gates**: 
    - You MUST run `./scripts/agent-exec.sh verify lint` and ensure it passes.
    - You MUST run `./scripts/agent-exec.sh verify coverage` to ensure no coverage regression.

## Scripts
- Default: `scripts/critique.sh`
- Analysis: `scripts/refactor.sh`
