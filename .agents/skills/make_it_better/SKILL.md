# Skill: Make It Better (Refactor)
name: make_it_better
description: |
  Optimize, clean, and DRY the working codebase without breaking the existing test suite.
  A 10x refactor MUST result in lower or equal cognitive complexity (gocognit)
  and zero new duplication (dupl).
---
## Objective
As the BackendEngineer, refactor the Green-phase implementation into its final 10x state.

## Rules of Engagement
- **The Specification**: You MUST read and address every item in `REF_TODO.md` (generated by the ChiefCritic).
- **The Red-Green Guard**: Do NOT change test inputs/outputs or functional behavior. Tests must stay green.
- **Complexity Guard**: You MUST run `scripts/refactor.sh` to verify that cognitive complexity did not increase.
- **DRY Policy**: No code blocks >= 15 lines should be duplicated. Extract to internal utilities.

## Scripts
- Default: `scripts/refactor.sh`
- Analysis: `scripts/refactor-drift.sh`
